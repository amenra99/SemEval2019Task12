Issuse:
내가 해결하고 싶은 이슈는 '유사 도메인(주제, 타입)에 있는 문서들을 어떻게 효율적으로 표현할 것인가' 이다.
파일시스템이나 검색엔진에서 보여주는 결과에서 문서는 보통 리스트 형태로 표현된다.
하지만 리스트 뷰는 제목이나 약간의 서머리를 통해 문서 내용을 유추해야 하기 때문에 많은 양의 목록을 확인할 때 적합하지 않다.
문서간 관계를 시각적으로 표현한다면 한꺼번에 여러 문서를 처리해야 하는 오버헤드를 줄이고 이해도를 높일 수 있지 않을까?
The issue I want to solve in this project is how to efficiently express multiple document relationship in a similar domain. Documents are usually shown in the form of lists in file systems or search engine results.
However, list view is not suitable for viewing large amounts of documents. Although it provides document titles and summaries, it is still difficult to idintify relationship between documents at once unless one reads every document. I think document relationship visualization can be an answer to solve this problem that can improve understanding of multiple documents.


Visualization Problem:
내가 이 이슈에 대해 시각적으로 해결하고 싶은 문제는 2가지 이다.
1) 문서간 관계를 어떻게 표현할 것인다? 문서간 관계는 네트워크 구조로 표현될 수 있다. 그러나 나는 기존의 네트워크 시각화 외에 문서가 가지는 고유한 특성을 특징적으로 표현할 수 있는 시각화 방법을 이 프로젝트의 결과로 얻기를 원한다.
문서와 같은 비정형데이터는 순위/속성을 정의하기가 쉽지 않지만, 나는 기본적으로 문서 내에 포함된 인물/기관/장소에 대하여 연결할 것이다. 추가로 문서간 유사도, 문서가 만들어진 시간에 따른 (타임라인), 사용자 입력 검색어 등도 적용되어야 할 것이다.
2) 레퍼런스 관리 툴이기 떄문에 사용자가 정의한 컨텐트(노트, 메모, 아이디어 등)도 시각화 범위에 포함되어야 한다. 다만, 이런 사용자 컨텍스트는 문서가 내포한 속성(NER)과는 구분되도록 표현되어야 한다. 특히, 사용자 컨텐트는 태그(키워드)나 카테고리(레이블) 뿐만아니라, 문장 또는 문단 이상의 비정형 구조를 가질 수도 있기 때문에 기존의 문서관계를 어지럽게하지 않으면서 readable하도록 표현할 필요가 있다. 나는 이 부분을 단순히 팝업하거나 다른 프레임에 문단을 출력하는 단순한 방법으로 해결하고 싶지는 않다.

There are two visualization problems that I would like to solve on this issue.
1) How to describe the relationship between documents? Document relationships can be expressed in a network topology. But, I would like to develop a novel visualization as a result of this project that represents unique characteristics of a document rather than traditional network visualizations. Basically, I want to show document relationships as linking people, organizations, location among documents. In addition, document similarity and timeline need be visualized as well.
2) How to describe user-defined content differently from NER attributes? Because it is a reference management tool, user-defiened contents, such as notes, memos, or outer links, should also be included in the visualization scope. In particular, user contents may have unstructured formats like sentences or paragraphs as well as keywords(tags) and labels(categories). User contents should be readable distinguishing from existing document relationships. I don't want to solve this by simply poping up or showing on the other frame.


Data:
추출 정확도, 문서간 유사도 등은 추후 별도의 알고리즘을 통해 정의되어야 하는 부분이다. 지금 시점에 알고리즘 정확도를 판단할 수 없기 때문에, 이 프로젝트를 위해서는 결과가 정해져있는 데이터를 사용하기를 희망한다. 그래서 NER 학습에 자주 사용되는 CoNLL 2012 데이터를 사용할 것이다. CoNLL 데이터는 뉴스, 웹 등 2,400개 문서에 사람이 분류한 인물/기관/장소가 정의되어 있다.
Since NER extraction accuracy or document similarity algorithms cannot be determined at this time, I want to use data with fixed results for the project. So, CoNLL 2012 data, which is often used for NER training, will be used. CoNLL data consist of about 2,400 documents including news, magazines, and web articles with manually annotated NER tags.


Extraction accuracy, inter-document similarity, etc. should be defined later through a separate algorithm. Since the algorithm accuracy cannot be determined at this time, we hope to use the data with fixed results for this project. So CoNLL 2012 data, which is often used for NER learning, will be used. CoNLL data are defined in 2,400 documents, including news, the Web, and people/organizations/location classified by people.
